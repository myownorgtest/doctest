Properties
Name Type Argument Description
hostNames array <optional>

Required. The fully qualified domain name for website. Only one hostname can be specified in the azurewebsites.net domain. The hostname should match the name of the website. Custom domains can only be specified for Shared or Standard websites.
name string

Required. The name of the website. This should match the hostname in the default domain.
computeMode WebSiteComputeMode <optional>

This value should be Shared for the Free or Paid Shared offerings, or Dedicated for the Standard offering. The default value is Shared. If you set ComputeMode to Dedicated, you must specify a value for the ServerFarm element.

The name of the Server Farm associated with this website. This is a required value for Standard mode.
siteMode WebSiteMode <optional>

This value is limited for the Free offering Basic for the Paid Shared offering. The default value is Limited. Note: Standard mode does not use the SiteMode setting; it uses the ComputeMode setting. For more information, see Upgrade or Downgrade a Web Site.

update(webSpaceName, webSiteName, parameters, callback) → {Stream}

You can update the settings for a web site by using the HTTP PUT method and by specifying the settings in the request body. (see http://msdn.microsoft.com/en-us/library/windowsazure/dn167005.aspx for more information)

Parameters:
Name Type Description
webSpaceName string

The name of the web space.
webSiteName string

The name of the web site.
parameters WebSiteUpdateParameters

Parameters supplied to the Update Web Site operation.

Properties
Name Type Argument Description
availabilityState WebSpaceAvailabilityState <optional>

The state of the availability of management information for the site. Possible values are Normal or Limited. Normal means that the site is running correctly and that management information for the site is available. Limited means that only partial management information for the site is available and that detailed site information is unavailable.
computeMode WebSiteComputeMode <optional>

The Compute Mode for the web site. Possible values are Shared or Dedicated.
enabled boolean <optional>

true if the site is enabled; otherwise, false. Setting this value to false disables the site (takes the site off line).
hostNameSslStates array <optional>

SSL states bound to the website.
hostNames array <optional>

An array of strings that contains the public hostnames for the site, including custom domains. Important: When you add a custom domain in a PUT operation, be sure to include every hostname that you want for the web site. To delete a custom domain name in a PUT operation, include all of the hostnames for the site that you want to keep, but leave out the one that you wangt to delete.
runtimeAvailabilityState WebSiteRuntimeAvailabilityState <optional>

Possible values are Normal, Degraded, or NotAvailable. Normal: the web site is running correctly. Degraded: the web site is running temporarily in a degraded mode (typically with less memory and a shared instance.) Not Available: due to an unexpected issue, the site has been excluded from provisioning. This typically occurs only for free sites.
sslCertificates array <optional>

SSL certificates bound to the web site.
serverFarm string <optional>

String. If a server farm exists, this value is DefaultServerFarm.
siteMode WebSiteMode <optional>

String that represents the web site mode. If the web site mode is Free, this value is Limited. If the web site mode is Shared, this value is Basic. Note: The SiteMode value is not used for Reserved mode. Reserved mode uses the ComputeMode setting.
state WebSiteState <optional>

A string that describes the state of the web site. Possible values are Stopped or Running.
